HTML Syntax

The grammatical rules that HTML follow.

File type

To create a webpage, we just need to create an ordinary text file. But instead of .txt, we'll change its filename extension into .html.

The HTML filename extension

When HTML was young, during a time when storage computers were really small, it was important to have really short filenames to give more space to actual file contents. In fact, Microsoft Windows went to the extreme and forced the three-character limit to filename extensions. This is why HTML files used to have .htm as its filename extension. Now, however, we have storage capacities that reach up to terabytes. The lengths of filenames aren't really an issue nowadays, and majority of websites use the longer .html filename extension. You can use any of these.

Syntax

This is what HTML looks like:

<p>

In this example, p means paragraph. To create this HTML, we type a less-than symbol (<) followed by a letter, word, or abbreviated word, and a greater-than symbol (>). We call this an HTML tag. This is what HTML markup looks like. HTML syntax is simple. Knowing which tag to use when is what's harder.

There are two types of tags: the closing and the opening tags. Our previous example <p> is an opening tag. This is what a closing tag looks like: 

</p>

Opening and closing tags look the same, except closing tags have a forward slash (/) after the less-than symbol (<).

To markup a paragraph, we put our text between the opening and closing tags:

<p>This is a paragraph</p>

Closing and opening tags are always together. When they are combined along with the content, we call it an element

Not all elements have a closing tag, however, like what we will see in future lessons. Most of the HTML elements have closing tags, though. Let's look at other examples:

<p>This is a paragraph.</p>
<h1>This is a headline.</h1>
<article>This is an article.</article>

We will look at many HTML elements throughout this class. We will study how each element is used and when. So don't worry about what each of these elements mean just yet. For now, let's focus on HTML's basic mechanics.

Indicating purpose and meaning is what markup is made for. It gives computers more details about what content or user interface means. HTML serves as a connection between the real world and the computer's perception of it. Let's look at another example.

<p>This is a paragraph. It contains text that is <em>emphasized</em> so reader would know that it is important.</p>

The whole paragraph is wrapped in <p></p> tags. Inside the paragraph, a word is wrapped in opening and closing em tags. You will learn what the em tag is for in the next lesson. For now, notice how we can put an HTML element inside another HTML element. This is what we call nesting. And as you will see in the next lessons, the HTML page is a big collection of HTML elements that are nested inside one another.

If you have a background in Computer Science, you will recognize that from this nesting pattern, we can build a structure called a document tree. This document tree is just like a family tree, with parents, children, and siblings. The browser is aware of this document tree and builds its own family tree based on which element is related to another and how. This is what we call the Document Object Model.

Ang Document Object Model (DOM) Tree ay isang hierarchy, isang structure ng HTML elements na kadalasan nang ginagamit para i-connect ang HTML sa CSS at JavaScript.

You will encounter this DOM tree when you are writing CSS and JavaScript.

The browser will then use this DOM tree to build another structure called the accessibility tree. We will also talk about accessibility later in this class. But for now, just remember that how you arrange and nest HTML elements inside one another and which HTML elements you choose to use will influence how your users will experience your website—including persons with disability. All of that starts with proper nesting of HTML elements. This next example is an HTML markup that has a nesting that you will use often in the future:

<article>
  <h1>This is a headline.</h1>
  <p>What you are reading is the first paragraph.</p>
  <p>That paragraph will be followed by this, the second one.</p>
  <p>This is a paragraph. May text dito na <em>naka-emphasize</em> para sa malaman ng reader na mahalaga ito.</p>
</article>
